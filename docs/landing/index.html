good night, have fun

Tell your agents good night.
Wake up to clean commits.

gnhf is the agent-agnostic orchestrator that keeps your coding agents running while you sleep. Each iteration ships a small, committed, documented change towards your objective. You wake up to a branch you can review, cherry-pick, or merge.

Get started Star on GitHub 
$ npm install -g gnhf
$ cd your-repo
$ gnhf "reduce complexity without changing functionality"

Why gnhf

>

Dead simple

One command starts an autonomous loop that runs until you Ctrl+C or hit a configured cap.

~

Long running

Commit on success, roll back on failure, exponential backoff on hard errors. Built for unattended overnight runs.

*

Agent-agnostic

Works with Claude Code, Codex, Rovo Dev, OpenCode, or GitHub Copilot CLI. Switch with a single flag.

&

Parallel worktrees

Run multiple agents on the same repo without interference. Each gets its own isolated working directory and branch.

$

Cost visibility

gnhf stats aggregates per-agent token spend and estimates the dollar cost of every overnight run.

!

Operator controls

Pause, steer, or abort a running loop without killing the process. Auto-PR via the gh CLI when commits land.

How it works

$ gnhf "add tests for the orchestrator module" --max-iterations 8
  iteration 1: read notes.md, plan first test
  iteration 2: add fixtures and run setup
  iteration 3: cover happy-path orchestrator loop
  iteration 4: cover token-cap abort
  ...
  ----------------------------------------------
  Run summary: 8 iters (7 ok / 1 failed) — $0.34
  Branch:      gnhf/add-tests-for-the-orchestra…
  Open PR:     https://github.com/you/repo/pull/123

gnhf creates a dedicated branch, writes prompt.md and shared notes.md, and loops: build the iteration prompt → invoke your agent → commit on success or roll back on failure → repeat. Token caps, iteration caps, and natural-language stop conditions keep runs bounded.
